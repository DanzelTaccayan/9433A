Before the World Wide Web :

WAIS
A system that enables client to search and retrieve documents in a central database. Unlike the WWW, WAIS does not link related documents together, instead documents are stored in databases where you have to query the document/s you want to retrieve.

Hypertext Transfer Protocol (HTTP)

Standardization: W3C and IETF

Underlying protocol used by the World Wide Web and this protocol defines how message are formatted and transmitted.

Runs on top of TCP/IP

Higher layer protocol that takes advantage of TCP/IP. Example: Packet Switching

Stateless communication protocol : servers dont keep information about clients in between requests.

Based on a client server architecture

Uses a request-response standard protocol. Example: The client sends an HTTP request message to the server, the server processes the request and replies with an HTTP response message

Uses the Port Number 80 which is the default

Port Number
A number from 0 to 65,635
Application uses this to uniquely identify each other. Example:If App W on IPA wants to send a message on App X on IPC on the same hand, App C on IPA wants to send a message on AppB on IPC, Port numbers makes this possible
IANA: the ones responsible for the allocation of certain ports
Socket
Consists of an IP address and a port number

Two Connections:
Non - Persistent Connections
Only available on HTTP 1.0 or HTTP 0.9, follows 4 steps:
  1. Connection
  2. Request
  3. Response
  4. Disconnection
A cycle because in requesting you need to do the 4 steps above, the cycle must be first completed before requesting another request.
Solution to Non-Persisten Connection are Persistent Connection
Persistent Connection:
Your request would first send an info about your request which would make the connection open for a brief of time
Available in HTTP 1.1
Makes use of Request Pipelining: First and second request can be sent one after another you dont need to wait until a request is finished not like in Non Persistent connection.

Things To Consider:

  • HTTP uses a request-response standard protocol. If the client sends an HTTP request message to the server, the server processes the request and replies with an HTTP response message
  • A server can also be a client. A suitable example would be: A client requests from a proxy server , the proxy server(The server which would now become a client) will then go and requests from a gateway.
  • Client always pulls from server by request(by using the "PULL" protocol"): client -> http request message -> server -> http response message -> client
  • Since HTTP is a stateless protocol, servers doesnt keep info about you it is possible to keep track of you actions in some webpages(E.g. Shopping sites) because of server side scripting technologies.
  • Server doesnt initiate request but with the PUSH protocol in HTTP 2.0 it is possible
HTTP 2.0 - "PUSH" protocol
Now the server can volunteer/initiate requests E.g. Facebook's news feed
