What is a network?

Interconnected devices that communicate with each other (to share resource)

A network is characterized by different nodes

Internet:
Networks connected to other networks globally, that is why it is also called "networks of networks".
Intranet
Networks connected to other networks within a private area.

Examples: Computer, Laptop, Desktop, Routers, Switch, Hub

Interconnection technology

Protocols:

Certain set of rules to follow

Example: Computers using serial communication(a bit per transmission) cannot send their transmission at the same time that is why protocols exist to fix the flow

Put source and its destination: Where to give and to receive

Send Big files: Chopping them into smaller pieces which is the work of protocols

Before the World Wide Web :

WAIS
A system that enables client to search and retrieve documents in a central database. Unlike the WWW, WAIS does not link related documents together, instead documents are stored in databases where you have to query the document/s you want to retrieve.
An Example would be a document (Document_1) you are reading is related to another document (Document_2), in order to get Document_2 you have to query that document in order to retrieve it.
Gopher
A protocol that is used for searching and retrieving documents. It uses Indexes to Subindexes in order to link other documents together("Hierarchies")
It was termed Gopher because like the animal Gopher it burrows from document to document until it reaches its destination.
Usenet
It is like a "Bulletin Board" or a "Forum". Its modern correspondence would be StackOverflow, and Reddit.

Hypertext Transfer Protocol (HTTP)

Standardization: W3C and IETF

Underlying protocol used by the World Wide Web and this protocol defines how message are formatted and transmitted.

Runs on top of TCP/IP

Higher layer protocol that takes advantage of TCP/IP. Example: Packet Switching

Stateless communication protocol : servers dont keep information about clients in between requests.

Based on a client server architecture

Uses a request-response standard protocol. Example: The client sends an HTTP request message to the server, the server processes the request and replies with an HTTP response message

Uses the Port Number 80 which is the default

Port Number
A number from 0 to 65,635
Application uses this to uniquely identify each other. Example:If App W on IPA wants to send a message on App X on IPC on the same hand, App C on IPA wants to send a message on AppB on IPC, Port numbers makes this possible
IANA: the ones responsible for the allocation of certain ports
Socket
Consists of an IP address and a port number

Versions of HTTP

  • HTTP 0.9 (1991) - The Standard HTTP
  • HTTP 1.0 (RFC 1945, May 1996)
  • HTTP 1.1 ( RFC 2068 Jan 1999, RFC 2616 Jun 1999),
    ( RFC 7230 - Message Syntax and Routing, RFC 7231 - Semantics and Content, RFC 7232 - Conditional Requests, RFC 7233 - Range Requests, RFC 7234 - Caching, RFC 7235 - Authentication Jun 2014)
  • HTTP 2 (RFC 7540 May 2015)

Two Connections:
Non - Persistent Connections
Only available on HTTP 1.0 or HTTP 0.9, follows 4 steps:
  1. Connection
  2. Request
  3. Response
  4. Disconnection
A cycle because in requesting you need to do the 4 steps above, the cycle must be first completed before requesting another request.
Solution to Non-Persisten Connection are Persistent Connection
Persistent Connection:
Your request would first send an info about your request which would make the connection open for a brief of time
Available in HTTP 1.1
Makes use of Request Pipelining: First and second request can be sent one after another you dont need to wait until a request is finished not like in Non Persistent connection.

Things To Consider:

  • HTTP uses a request-response standard protocol. If the client sends an HTTP request message to the server, the server processes the request and replies with an HTTP response message
  • A server can also be a client. A suitable example would be: A client requests from a proxy server , the proxy server(The server which would now become a client) will then go and requests from a gateway.
  • Client always pulls from server by request(by using the "PULL" protocol"): client -> http request message -> server -> http response message -> client
  • Since HTTP is a stateless protocol, servers doesnt keep info about you it is possible to keep track of you actions in some webpages(E.g. Shopping sites) because of server side scripting technologies.
  • Server doesnt initiate request but with the PUSH protocol in HTTP 2.0 it is possible
HTTP 2.0 - "PUSH" protocol
Now the server can volunteer/initiate requests E.g. Facebook's news feed
Polling
is the continuous checking of other programs or devices by one progam or device to see what state they are in, usually to see whether they are still connected or want to communicate.
works with pull but not efficient because it wastes of bandwidth and process cycles.

Request Header Fields

Name Description
Accept this is used to specify a certain media types and to indicate a request with a limitation in a set of types.
Accept-charset his is used to specify the character sets than can be used for a response.
Accept-encoding Same as the accept request header field, but this restricts the content-codings which are acceptable in a response.
Accept-Language This restricts the set of natural languages that was choosed as the response to a request. This is also same as the accept request header field.
Expect This is used to indicate that specific server behaviors that are required by a client.
From If this is given, it must contain an internet e-mail which is addressed for the human user who is controlling the request user agent.
Host This will indicate the internet host and the port number of the resource which is being requested.
If-Match This is used and is accompanied with a method to make it conditional. Client with one or more entities that is previously obtained from a resource can be verified that one of the entities is current.
If-Modified-Since This is used and is accompanied with a method to make it conditional: If the variant that is requested hasn't been modified yet since the moment it was specified in the field, an entity will not be returned to the server.
If-Not-Match This is used and is accompanied with a method to make it conditional. The purpose of If-None-match is to have and allow an efficient updates of cached information.
If-Range This will allow a client to have a 'Short-circuit' to the second rquest.
If-Unmodified-Since This is used and is accompanied with a method to make it conditional. If those variants requested was not modified since the moment it was in the field, an entity will not be returned from the server.
Max-Forward This is used to provide a mechanism with the trace and options methods which will limit the quantity of proxies or gateways.
Proxy-Authorization This is used to allow a client identify itself to a proxy.
Range two types: Range Retrieval Requests and Byte Ranges
Referer This will permit the client to indentify the address of the resource where we can obtain the request-URI.
User-Agent This holds the informations originating the request about the user agent.

RESPONSE HEADER FIELDS

Name Description
Accept-Range This will auhorize a server to indicate its acceptance of range requests for a resource.
Age This is used to convey the sender's estimated time from the moment the response was generated to an origin server.
E-tag This will give you the current value of the entity-tag for the requested variant.
Location This is used to redirect those recipients to other location aside from the request-URI.
Proxy-Authenticate This is should always be included as part of a 407 response.
Retry-After This is used to a 503 response to tell on how long will it take the service to be unavailable to the request client.
Server This holds the information to the software that has been used to handle the request by the origin server.
Vary This will specify the set of request-header fields that will fully determine whether a cache is allowed to use response to a subsequent request without revalidation.
WWW-Authenticate This should be included in a 401 response message.

ENTITY HEADER FIELDS

Name Description
Allow This will enumarate all the set of methods that are allowed and supported by the resource indentified by the request-URI.
Content-Encoding This is used as the modifier to a media-type.
Content-Language This is used to describes the natural langauages which is intended to use to audiences for the enclosed entity.
Content-Length This will show the size of the entity-body.
Content-Location This might be used to supply the resource location for the entity enclosed in a message.
Content-MD5 "This is an MD5 digest of an entity-body for the purpose of providing an end-to-end message integrity check (MIC) of the entity-body" as defined in RFC 1864[23].
Content-Range This is sent with a partial entity-body which is used to specify where to put and apply in the full entity-body this partial body.
Content-Type This will show the media type of the entity-body that was sent to a recipient.
Expires This will provide you the date and time after a response is considered stale.
Last Modified This will show the date and time to which the origin server was believed the variant was last modified.
