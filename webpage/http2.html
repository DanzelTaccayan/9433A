HTTP provides support for other functionalities such as:

Cache Control
Stores certain things that has already been accessed on a storage location and then retrieves it when it is needed.
It's foolish to refetch the same content when accessing the same website on different webpages so cache is there to make access fast.
Content Media Type(MIME) Specification
Web uses MIME to identify what type of media it is so that the other end could know how to process it.
Language and Character Set Specification
Protocol handles it
Content/Transfer Encodings
Sending Files/Text. Example: Sending Compressed files
Content Negotiation
Servers should know how to handle files that are being sent. Example: A compressed file should be decompressable
Server and Client must agree
Client Server Protocol negotiations
Two Connections:
Non - Persistent Connections
Only available on HTTP 1.0 or HTTP 0.9, follows 4 steps:
  1. Connection
  2. Request
  3. Response
  4. Disconnection
A cycle because in requesting you need to do the 4 steps above, the cycle must be first completed before requesting another request.
Solution to Non-Persisten Connection are Persistent Connection
Persistent Connection:
Your request would first send an info about your request which would make the connection open for a brief of time
Available in HTTP 1.1
Makes use of Request Pipelining: First and second request can be sent one after another you dont need to wait until a request is finished not like in Non Persistent connection.
