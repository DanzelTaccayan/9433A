Hypertext Transfer Protocol (HTTP)

Hypertext Transfer Protocol (HTTP)

Things To Consider:

  • HTTP uses a request-response standard protocol. If the client sends an HTTP request message to the server, the server processes the request and replies with an HTTP response message
  • A server can also be a client. A suitable example would be: A client requests from a proxy server , the proxy server(The server which would now become a client) will then go and requests from a gateway.
  • Client always pulls from server by request(by using the "PULL" protocol"): client -> http request message -> server -> http response message -> client
  • Since HTTP is a stateless protocol, servers doesnt keep info about you it is possible to keep track of you actions in some webpages(E.g. Shopping sites) because of server side scripting technologies.
  • Server doesnt initiate request but with the PUSH protocol in HTTP 2.0 it is possible
