Request Header Fields

Name Description
Accept this is used to specify a certain media types and to indicate a request with a limitation in a set of types.
Accept-charset his is used to specify the character sets than can be used for a response.
Accept-encoding Same as the accept request header field, but this restricts the content-codings which are acceptable in a response.
Accept-Language This restricts the set of natural languages that was choosed as the response to a request. This is also same as the accept request header field.
Expect This is used to indicate that specific server behaviors that are required by a client.
From If this is given, it must contain an internet e-mail which is addressed for the human user who is controlling the request user agent.
Host This will indicate the internet host and the port number of the resource which is being requested.
If-Match This is used and is accompanied with a method to make it conditional. Client with one or more entities that is previously obtained from a resource can be verified that one of the entities is current.
If-Modified-Since This is used and is accompanied with a method to make it conditional: If the variant that is requested hasn't been modified yet since the moment it was specified in the field, an entity will not be returned to the server.
If-Not-Match This is used and is accompanied with a method to make it conditional. The purpose of If-None-match is to have and allow an efficient updates of cached information.
If-Range This will allow a client to have a 'Short-circuit' to the second rquest.
If-Unmodified-Since This is used and is accompanied with a method to make it conditional. If those variants requested was not modified since the moment it was in the field, an entity will not be returned from the server.
Max-Forward This is used to provide a mechanism with the trace and options methods which will limit the quantity of proxies or gateways.
Proxy-Authorization This is used to allow a client identify itself to a proxy.
Range two types: Range Retrieval Requests and Byte Ranges
Referer This will permit the client to indentify the address of the resource where we can obtain the request-URI.
User-Agent This holds the informations originating the request about the user agent.
