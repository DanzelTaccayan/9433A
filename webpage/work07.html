Response Header Fields

Name Description
Accept-Range This will auhorize a server to indicate its acceptance of range requests for a resource.
Age This is used to convey the sender's estimated time from the moment the response was generated to an origin server.
E-tag This will give you the current value of the entity-tag for the requested variant.
Location This is used to redirect those recipients to other location aside from the request-URI.
Proxy-Authenticate This is should always be included as part of a 407 response.
Retry-After This is used to a 503 response to tell on how long will it take the service to be unavailable to the request client.
Server This holds the information to the software that has been used to handle the request by the origin server.
Vary This will specify the set of request-header fields that will fully determine whether a cache is allowed to use response to a subsequent request without revalidation.
WWW-Authenticate This should be included in a 401 response message.
